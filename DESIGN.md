## Phase 1: Streamlit POC

### How to import the calculation module

`graha_positions_reference.py` has an `if __name__ == "__main__":` guard and
exposes `calculate_graha_positions_for_local_dt(local_dt, loc)` at module level.
It is directly importable. Use `pathlib` to locate it relative to `app.py`:

```python
import sys
from pathlib import Path

# scripts/ is a sibling of streamlit/, so go up one level
sys.path.insert(0, str(Path(__file__).parent.parent / "scripts"))
from graha_positions_reference import (
    calculate_graha_positions_for_local_dt,
    DEFAULT_LOCATION,
)
```

### Ephemeris data

`graha_positions_reference.py` does **not** call `swe.set_ephe_path()`. It uses
`pyswisseph`'s built-in Moshier ephemeris, which is bundled inside the Python
package itself. No `.se1` files need to be copied into `streamlit/`. The app
is self-contained.

### Auto-refresh for real-time updates

Use the `streamlit-autorefresh` package (not a built-in Streamlit function):

```python
from streamlit_autorefresh import st_autorefresh
st_autorefresh(interval=60_000, key="planet_refresh")  # refresh every 60s
```

Add `streamlit-autorefresh` to `requirements.txt`.

### Location input

Streamlit Community Cloud does not support custom JS components out of the box,
so browser geolocation is not available in Phase 1. Default to Ujjain
(23.18°N, 75.78°E) with a `st.sidebar` lat/lon/timezone input the user can
override:

```python
with st.sidebar:
    lat = st.number_input("Latitude",  value=23.1828, format="%.4f")
    lon = st.number_input("Longitude", value=75.7772, format="%.4f")
    tz  = st.text_input("Timezone",    value="Asia/Kolkata")
```

### Four views in Streamlit

| View | Matplotlib approach |
|---|---|
| Rashi Wheel | `ax = fig.add_subplot(projection='polar')` — 12 rashi wedges at 30° each, Aries at top, counter-clockwise. Set `ax.set_theta_zero_location("N"); ax.set_theta_direction(-1)` to place 0° at the top (Matplotlib defaults to 3 o'clock, which is wrong). Planet dots at their sidereal longitude. |
| Celestial Sphere | Scatter plot: RA (x-axis, reversed) vs Dec (y-axis). 9 labelled markers. |
| North Kundali | `ax.broken_barh` or patches to draw the 12-box diamond grid. |
| South Kundali | Grid of 12 fixed rectangular cells using `matplotlib.patches.Rectangle`. |

Rashi at 0°: Aries (Mesha). Direction: counter-clockwise (standard Vedic convention,
matching the iOS app).

## Phase 2: React Web App (specced separately)

High-level direction only — full spec written when Phase 1 ships.

- **Frontend**: React + TypeScript + Vite. Hosted on **GitHub Pages**
  (`yourname.github.io/jyotish_skyview`). Free, no credit card, HTTPS included.
- **"Backend" (no server required)**: A **GitHub Actions cron job** runs
  `scripts/graha_positions_reference.py` every 5 minutes, commits the output
  as `web/public/positions.json` to the repo. GitHub Pages serves the file.
  The React app polls that URL on an interval. Total cost: $0, no credit card,
  works for public repos.
- **Alternative backend**: PythonAnywhere free tier — genuine free, no credit
  card, runs a FastAPI or Flask app at `yourname.pythonanywhere.com`. Good
  fallback if cron-based positions feel too stale.
- **Avoid**: Vercel Python Serverless Functions (pyswisseph is a C extension,
  won't compile in Vercel's build environment), Render.com (free tier removed
  2023), Railway (requires credit card), Fly.io (requires credit card).

Effort estimate: 4-6 weeks for someone learning React from scratch (React state
management + Canvas 2D for the animated Rashi wheel is non-trivial).

  the fixture JSON)

- A friend can open the URL without installing anything

### Phase 2 done when:
- React app is live on Vercel
- Rashi wheel animates in real time (positions update every 60s via API poll)
- All 4 views work on mobile (responsive)
- Link is publicly shareable

## The Assignment

Before writing a single line of `streamlit/app.py`, run this in a local terminal:

```bash
cd scripts
python - <<'EOF'
from graha_positions_reference import calculate_graha_positions_for_local_dt, DEFAULT_LOCATION
from datetime import datetime
from zoneinfo import ZoneInfo
dt = datetime.now(ZoneInfo(DEFAULT_LOCATION["timezone"]))
result = calculate_graha_positions_for_local_dt(dt, DEFAULT_LOCATION)
for g in result:
    print(f"{g['graha']:10s}  {g['sidereal_lon']:.4f}°  {g['rashi']}")
EOF
```

If that prints 9 planet positions cleanly, the import works. Then start the
Streamlit app from that exact import and build outward.
